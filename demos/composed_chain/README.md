# Composed Chain Demo

This demo shows how to build a staged composition with
`ComposedFunction.chain(...)`.

The composition uses three heterogeneous stages:

$$G_1(a) =
\begin{bmatrix}
a_1 + 1 \\
0.5 a_2 + 2
\end{bmatrix}, \qquad
G_2(a, p^{(1)}) =
\begin{bmatrix}
a_1 + p^{(1)}_1 a_2 \\
a_2 + p^{(1)}_2
\end{bmatrix}, \qquad
G_3(a, p^{(2)}) =
\begin{bmatrix}
0.25 a_1 + p^{(2)}_1 \\
2 a_2 - p^{(2)}_2
\end{bmatrix}.$$

The final state is

$$f(x, w) = G_3(G_2(G_1(x), p^{(1)}), p^{(2)}),$$

where the packed parameter vector is

$$w = \left(p^{(1)}, p^{(2)}\right).$$

This is a good minimal example because it shows:

- `chain([...])` for heterogeneous stage sequences
- a mixture of fixed and symbolic stage bindings
- one packed `parameters` input in the generated Rust code
- one Rust crate containing both the staged primal and staged Jacobian kernels
- direct code generation from a composed symbolic source

## Files

- [`main.py`](./main.py): builds the composed function and generates Rust
- `composed_chain_kernel/`: generated by running `main.py`
- `runner/`: a small Rust binary crate that depends on `composed_chain_kernel`
  by path, calls the staged primal and Jacobian kernels, and prints their
  outputs

## Running the demo

It is advisable to run the demo from an activated virtual environment.

From the repository root, after activating your virtual environment:

```bash
python demos/composed_chain/main.py
```

This will:

- build a composition that chains three heterogeneous stages
- evaluate the composed function in Python
- evaluate the composed Jacobian in Python
- generate one staged Rust crate in:

```text
demos/composed_chain/composed_chain_kernel
```

To see how the generated crate can be used from Rust in practice, run:

```bash
cd demos/composed_chain/runner
cargo run
```
