# Multi-Function Crate Demo

This demo shows how `CodeGenerationBuilder` can place several source
functions in the same generated Rust crate.

The example defines two scalar-valued functions of the same inputs
$x \in \mathbb{R}^2$ and $u \in \mathbb{R}$:

$$
f(x, u) = x_1^2 + x_2^2 + u_1 x_1
$$

and

$$
g(x, u) = x_1 x_2 + \cos(u_1).
$$

The point of the demo is not the algebra itself, but the code-generation
workflow:

1. define more than one `Function`
2. register each one with `CodeGenerationBuilder.for_function(...)`
3. request different generated kernels for each source function
4. build one Rust crate containing all of them

In this example:

- `f` contributes a primal kernel and Jacobian kernels
- `g` contributes a primal kernel, gradient kernels, and an HVP kernel

## Files

- [`main.py`](./main.py): defines the symbolic functions and generates Rust code
- `multi_function_kernel/`: generated by running `main.py`
- `runner/`: a small Rust binary crate that depends on `multi_function_kernel`
  by path, calls several generated functions from the shared crate, and prints
  their outputs

## Running the demo

It is advisable to run the demo from an activated virtual environment.

From the repository root, after activating your virtual environment:

```bash
python demos/multi_function/main.py
```

This will:

- print sample evaluations of both functions
- generate a Rust crate in:

```text
demos/multi_function/multi_function_kernel
```

To see how the generated crate can be used from Rust in practice, run:

```bash
cd demos/multi_function/runner
cargo run
```
