# Single Shooting Demo

This demo shows how to define a deterministic finite-horizon single-shooting
optimal-control problem and generate compact loop-based Rust code for:

- the total cost
- the gradient of the total cost with respect to the packed control sequence
- a joint kernel that returns cost, gradient, and rollout states together

The one-step dynamics is

$$f(x, u, p) =
\begin{bmatrix}
x_1 + p_1 x_2 + u \\
x_2 + p_2 u - 0.5 x_1
\end{bmatrix},$$

the stage cost is

$$\ell(x, u, p) = x_1^2 + 2 x_2^2 + 0.3 u^2 + p_1 u,$$

and the terminal cost is

$$V_f(x, p) = 3 x_1^2 + 0.5 x_2^2 + p_2 x_1.$$

For a fixed horizon length $N$, an initial state $x_0$, a packed control
sequence

$$u_{\mathrm{seq}} = (u_0, u_1, \dots, u_{N-1}),$$

and a shared parameter vector $p$, the states evolve according to

$$x_{t+1} = f(x_t, u_t, p), \qquad t = 0, \dots, N-1.$$

The total cost function is then

$$V_N(x_0, u_{\mathrm{seq}}, p)
=
\sum_{t=0}^{N-1} \ell(x_t, u_t, p) + V_f(x_N, p).$$

This is the main quantity we want to evaluate efficiently. In numerical
optimal control, we also want the gradient of this total cost with respect to
the entire input sequence,

$$\nabla_{u_{\mathrm{seq}}} V_N(x_0, u_{\mathrm{seq}}, p),$$

because first-order optimization methods use that gradient to update the whole
control trajectory.

The generated Rust kernel takes:

- `x0`: the initial state
- `u_seq`: the packed input sequence, laid out stage-major
- `p`: one shared parameter vector used by all stages and the terminal cost

The generated API can compute:

- the total cost only
- the gradient with respect to `u_seq` only
- both together
- and optionally the full rollout state trajectory

## Files

- [`main.py`](./main.py): builds the single-shooting problem and generates Rust
- `single_shooting_kernel/`: generated by running `main.py`
- `runner/`: a small Rust binary crate that depends on `single_shooting_kernel`
  by path, prints the generated metadata, calls the kernels, and prints their
  outputs

## Running the demo

It is advisable to run the demo from an activated virtual environment.

From the repository root, after activating your virtual environment:

```bash
python demos/single_shooting/main.py --horizon 5
```

This will:

- build the single-shooting problem
- evaluate the cost and gradient in Python
- generate one Rust crate in:

```text
demos/single_shooting/single_shooting_kernel
```

To see how the generated crate can be used from Rust in practice, run:

```bash
cd demos/single_shooting/runner
cargo run
```
