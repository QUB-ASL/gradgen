# Composed Function Demo

This demo shows how to build a staged composition with `ComposedFunction`.

The state update is

$$
G(a, p) =
\begin{bmatrix}
a_1 + p_1 \\
a_2 p_2
\end{bmatrix},
$$

and the terminal scalar function is

$$
h(a, p_f) = a_1 + a_2 + p_f.
$$

The composed function starts directly with $N$ repeated applications of the
same stage:

$$
f(x, w) = h(a_N, p_f),
$$

where

$$
a_0 = x, \qquad a_k = G(a_{k-1}, p^{(k)}) \quad \text{for } k = 1, \dots, N.
$$

Here the packed parameter vector is

$$
w = \left(p^{(1)}, p^{(2)}, \dots, p^{(N)}, p_f\right).
$$

This is a good minimal example because it shows:

- `repeat(...)` for repeated application of the same stage
- user-controlled repeat count through a command-line argument
- one packed `parameters` input in the generated Rust code
- staged gradient generation with reverse loops

## Files

- [`main.py`](./main.py): builds the composed function and generates Rust
- `composed_primal_kernel/`: generated by running `main.py`
- `composed_gradient_kernel/`: generated by running `main.py`

## Running the demo

It is advisable to run the demo from an activated virtual environment.

From the repository root, after activating your virtual environment:

```bash
python demos/composed_function/main.py --repeats 5
```

This will:

- build a composition where `G` is applied `N` times
- evaluate the composed function in Python
- evaluate the composed gradient in Python
- generate Rust crates in:

```text
demos/composed_function/composed_primal_kernel
demos/composed_function/composed_gradient_kernel
```
